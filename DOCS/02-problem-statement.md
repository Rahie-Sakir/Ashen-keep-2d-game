# Problem Statement

Many beginner game development projects tightly couple user interfaces with gameplay logic, making debugging and feature expansion difficult. As projects grow, maintaining such code becomes increasingly challenging.

Ashen Keep addresses this problem by implementing a modular architecture where gameplay logic, rendering, and user input are separated into independent components. This approach improves code quality, simplifies maintenance, and supports future development.

The project also serves as an educational example of applying software engineering principles to game development. Rather than focusing only on gameplay, it emphasizes clean architecture, maintainability, and documentation.

## Objectives

- Reduce coupling between software components.
- Improve code readability.
- Simplify testing and debugging.
- Demonstrate best practices in software engineering.
- Build a reusable architecture for future game features.
