# Interviews

## Objective

Informal interviews were conducted with fellow students and peers to gather feedback regarding gameplay expectations and software usability.

## Participant Profile

- Undergraduate computer science students
- Casual desktop game players
- Individuals familiar with Python applications

## Sample Questions

1. What makes a desktop game enjoyable?
2. How important are responsive controls?
3. Do users prefer simple or complex interfaces?
4. What problems are commonly experienced in student game projects?

## Summary of Feedback

### Responsive Controls

Participants emphasized that delayed or inconsistent controls negatively impact gameplay.

### Simple User Interface

Most users preferred intuitive interfaces that avoid unnecessary complexity.

### Stability

Participants valued applications that execute consistently without crashes.

### Expandability

Several interviewees suggested designing projects in a modular way so additional features can be added easily.

## Impact on Development

The interview findings influenced decisions such as:
- Modular architecture
- Dedicated input handling
- Separation of backend and frontend
- Emphasis on maintainable code
